================================================================================
SECTION 5: ROLLBACK INSTRUCTIONS
================================================================================

If something goes wrong and you need to undo the commit:

# Option 1: Revert (safe — creates a new commit that undoes the changes)
git revert HEAD
git push origin main

# Option 2: Reset to previous state (DESTRUCTIVE — rewrites history)
# WARNING: Only use if the bad commit hasn't been shared with anyone
git reset --hard HEAD~1
git push origin main --force-with-lease

# Option 3: Delete specific files and recommit
git rm filename-to-delete.md
git commit -m "chore: remove incorrectly uploaded file"
git push origin main


================================================================================
SECTION 6: ALTERNATIVE — HTTPS INSTEAD OF SSH
================================================================================

If you don't have SSH keys set up, use HTTPS with a personal access token:

# Step 1: Create a personal access token on GitLab:
# Go to: https://gitlab.com/-/user_settings/personal_access_tokens
# Name: "claudeprepfiles-upload"
# Scopes: read_repository, write_repository
# Copy the token immediately — you won't see it again

# Step 2: Configure git to use the token:
git config --global credential.helper store

# Step 3: Clone with HTTPS:
git clone https://github.com/Janisher-Illderment/claudeprepfiles.git
cd claudeprepfiles

# OR configure an existing clone:
git remote set-url origin https://github.com/Janisher-Illderment/claudeprepfiles.git

# Then proceed with the same git add / commit / push commands from Section 3.

# WARNING: Storing tokens in URLs can be a security risk.
# After upload, consider removing the token from the URL:
git remote set-url origin https://github.com/Janisher-Illderment/claudeprepfiles.git
